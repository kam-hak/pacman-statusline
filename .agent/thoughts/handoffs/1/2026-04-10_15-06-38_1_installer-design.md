## Learnings

- **Both patched and backup TTFs have U+F0BB0 in their cmap**, but the outlines differ at that codepoint (verified via fontTools `RecordingPen`). U+F0BAF (right-pac) outlines are byte-identical between patched and backup — that glyph is stock upstream Nerd Fonts, untouched. So the "patch" in the original handoff refers specifically to the left-pac glyph, and the upstream font already has *something* at F0BB0 that the user's patch replaces. The installer cannot safely reuse F0BB0 without clobbering whatever upstream put there — hence the design decision to scan for the next free PUA codepoint above U+F1000.
- **The original patching SVG is not on disk.** Searched `~/Documents`, `~/Downloads`, `~/Desktop`, grep'd for "F0BB0" content, searched for any `.sfd` or pac-named SVGs. Nothing. User confirmed: *"i reversed the glyph so it would go backwards. we just need to find that svg so that others can use it."* — but the reversal was likely done directly in FontForge (Element → Transform → Flip Horizontal → save font) without producing a standalone source file. **The SVG extracted from the live TTF is functionally equivalent** and can serve as the reference for reproducing the patch. Don't spend more time searching.
- **User correction on scope confusion**: when I mentioned "git" in the refactor context, user said *"i don't understand why you did all that git stuff.. this is a gist."* The confusion was that the statusline has a `GitStats` segment (for the current project's dirty state) which has nothing to do with where the statusline file itself is stored. Worth clarifying preemptively in future sessions that "the Git segment reads the cwd's repo, not the statusline's own storage."
- **User's collaboration style**: terse, casual, decisive. Doesn't want mechanical recaps. Says "good" and "MIT is fine" as full responses. When asked multiple questions, may answer only the one they care about and ignore the rest — be willing to pick sensible defaults on the unanswered ones.
- **"Install" needs plain-language explanation** for this user. They said *"idk what it even means to install this lol."* Don't assume familiarity with symlinks, PATH, `~/.claude/` semantics. The explanation I gave (one-liner + "replaces the file with a shortcut") landed well.
- **Autocompact buffer is 15% for 200k models, 17% for 1M**. The context quality-budget display in the statusline treats the degradation threshold at 350K tokens for 1M models, so ctx% can go *negative* past that. Matrix harness row `1M ctx=30% (deep zone)` shows `ctx:-100%` — that's correct, not a bug.
- **Bash 3.2 constraint**: macOS ships bash 3.2. The refactor uses prefixed vars (e.g., `W5_used`, `REPO_name`) rather than associative arrays. Don't regress this in the installer if it ends up needing any bash helpers.
